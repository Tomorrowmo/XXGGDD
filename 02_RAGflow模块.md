# 第二章 RAGflow 模块

## 2.1 模块概述

RAGflow 模块是导弹动力智能评估平台的智能交互入口，集成了大语言模型通用对话与检索增强生成知识库两大核心能力。该模块旨在为评估人员提供自然语言驱动的专业知识查询和数据分析辅助服务，将分散的领域知识资产转化为可实时调用的智能资源，有效支撑导弹动力系统方案论证与产品选型工作。

在航天动力系统评估实践中，评估人员经常需要查阅设计规范、历史试验报告、材料性能手册和故障案例库等大量专业资料。这些资料通常分散存储在不同的文档管理系统和个人积累中，检索效率低，且难以与当前分析任务形成有效关联。RAGflow 模块通过将大语言模型的语义理解能力与本地化部署的知识库检索系统深度集成，使评估人员能够在统一平台界面内以对话方式获取领域知识支持，显著降低信息获取成本，提升评估工作的知识密集度和专业一致性。

## 2.2 本地化部署与数据安全保障

RAGflow 知识库系统采用本地化部署方式，部署于平台所在的 GPU 服务器集群上，所有知识文档的存储、索引构建和检索推理均在甲方可控的硬件环境中完成，无需将任何数据上传至外部公有云服务。这一架构从根本上保障了导弹动力系统相关技术资料的数据安全性和保密合规性，满足军工领域对信息安全的基本要求。

本地化部署带来的另一核心优势是知识库的可持续积累与自主更新能力。随着评估工作的持续推进，每一次新的试验分析、每一次仿真验证、每一份评估报告都可以作为新的知识资产沉淀到 RAGflow 知识库中。知识库不是一次性交付的静态资源，而是一个能够伴随评估业务增长而持续进化的动态知识系统。例如，当新增某一型号的试验数据时，相关的性能参数基准和典型工况特征可以整理为结构化文档入库；当发现新的故障模式时，其现象描述、根因分析和处置措施可以补充到故障案例集中。通过这种"边用边建"的模式，知识库的覆盖范围和分析深度将随着使用时间的增长而不断提升，逐步构建起导弹动力领域专属的知识图谱。

在模型层面，平台当前通过 OpenAI 兼容接口同时支持本地部署的开源大模型和商业云端模型。随着知识库的持续积累和数据标注工作的推进，未来可基于积累的领域专用数据对开源基座模型进行微调，使模型在导弹动力领域的专业术语理解、技术规范引用和故障模式判别等方面的表现显著优于通用模型。经过微调的领域专用模型与持续更新的本地知识库相结合，将形成"专业知识检索 + 领域模型推理"的双重智能增强闭环——知识库为模型提供事实依据和标准参照，微调模型则具备更强的领域语义理解能力以更精准地利用知识库内容，两者相互促进，持续提升平台的智能分析水平。这一技术路线完全基于本地化基础设施实现，数据和模型资产始终保留在甲方可控范围内，不存在对外泄露风险。

## 2.3 多模型路由机制

该模块的技术实现核心是一套基于配置文件驱动的通用多模型路由机制。所有大语言模型的连接信息通过统一的模型配置文件进行管理，每个模型的配置项涵盖模型标识符、API 服务端点地址、具体模型名称、最大上下文长度、温度参数以及是否支持多模态等完整元信息。

当用户在前端界面中选择某一模型并发送消息时，后端服务根据模型标识符从配置文件中自动读取对应的连接参数，将请求路由到正确的 API 服务端点，整个过程对用户完全透明——评估人员只需在下拉菜单中选择模型名称即可，无需关心该模型由哪个服务提供、通过哪个端口访问。这种热插拔式的模型管理机制使得平台可以同时接入本地私有化部署的模型和商业云端模型，模型的新增、替换和下线均可通过配置管理界面完成，无需修改任何代码或重启服务。

## 2.4 前端交互设计

模块的前端界面采用"通用对话"与"知识库"两个子标签页的布局方式，评估人员可根据当前工作需求自由切换。

通用对话子标签提供标准的对话式交互环境。界面顶部设有模型选择工具栏，可用模型按"本地模型"和"商用模型"分组展示，方便评估人员快速定位目标模型。选择模型后可直接开始多轮对话，系统自动维护对话上下文。消息发送和接收采用对话气泡形式呈现——用户消息和 AI 回复分别以不同样式区分显示，AI 回复内容通过服务端推送事件流技术实现逐字实时输出，呈现打字机式的流式阅读体验，评估人员无需等待完整生成即可开始阅读。回复内容自动渲染 Markdown 格式，支持标题层级、数据表格、结构化列表和代码块等富文本元素，确保技术讨论中的参数表格、分析步骤等结构化信息呈现清晰易读。工具栏中还设有模型连接测试按钮，可一键发送轻量级探测请求验证 API 端点的可达性和响应延迟，帮助评估人员在正式使用前快速确认模型服务状态。

知识库子标签通过嵌入本地部署的 RAGflow 系统共享页面，为评估人员提供已整理好的领域知识文档检索服务。用户可以直接在嵌入的知识库页面中进行自然语言检索，定位相关的文档片段，也可将知识库页面在新浏览器窗口中独立打开使用。

## 2.5 知识库内容体系

RAGflow 知识库当前已收录与导弹动力系统相关的基础知识资料，涵盖以下核心类别：

（1）设计原理与基础理论，包括液体/固体火箭发动机工作循环、燃烧组织方式、喷管流动特性、推力矢量控制等基础技术原理；

（2）性能参数基准，包括不同推力级别发动机的典型室压范围、比冲参考值、特征速度、推力系数等关键性能参数的行业基准数据；

（3）典型故障模式案例，包括燃烧不稳定、喷管烧蚀、推进剂供应异常、点火失败等常见故障的现象特征、根因分析逻辑和处置措施；

（4）技术标准与规范摘要，包括航天动力系统设计、试验和验收相关国军标和行业标准的核心条款摘要；

（5）历史评估案例，包括过往项目中积累的乙方方案评估记录、关键指标的比对结论和评审意见。

以上知识内容将随着平台的持续使用和新项目的不断推进而动态更新和扩充。评估人员在日常工作中发现的值得归档的技术资料，均可按照知识库的内容模板整理后入库，逐步丰富领域知识资产的覆盖面和深度。

## 2.6 典型应用模式

RAGflow 模块在导弹动力智能评估场景中支持多种典型应用模式。

在方案理解阶段，评估人员首次接触某一乙方的动力方案时，可就方案中涉及的燃烧组织方式、喷管构型或推进剂类型等技术特点向 AI 提问，AI 结合知识库中的相关文献和基准数据给出专业的背景解读，帮助评估人员快速建立对该方案技术路线的整体认知框架，从而更有针对性地开展后续的量化分析。

在多方案横向比对阶段，评估人员可将各方案的关键技术特征和性能参数以自然语言描述的方式提交给 AI，由 AI 辅助梳理各方案之间的技术差异点和相对优劣势分布，识别出各方案在哪些性能维度上具有竞争优势、在哪些维度上可能存在短板，为后续的深度量化比对和专家评审提供有价值的参考线索。

在报告编制阶段，评估人员可以就特定技术术语的标准定义、性能参数的行业基准范围、故障模式的规范分类方式等问题向 AI 咨询，确保评估报告中的专业表述符合行业惯例和技术规范，提升报告的专业性和权威性。同时，AI 还可以辅助校对报告文本的逻辑连贯性和术语一致性。

## 2.7 通用性与扩展性

值得说明的是，RAGflow 模块的通用对话功能并不局限于航天动力领域。由于模型配置的高度灵活性和接口的标准化设计，该模块本质上是一个通用的 AI 对话工作台。评估人员可以根据实际需要接入任何符合 OpenAI 接口规范的大语言模型，将其应用于代码辅助开发、技术文档撰写、试验数据初步解释等跨领域的智能辅助任务。这种开放性设计确保了平台在甲方 IT 环境中的长期可用性和技术演进空间，不会因为特定模型服务的变更或升级而影响平台的正常运行。
