# 第五章 后续开发计划

## 5.1 总体路线概述

导弹动力智能评估平台的开发遵循"夯实基础、逐层递进、持续演进"的总体策略，按照功能成熟度和技术复杂度将后续工作划分为近期、中期和远期三个阶段。近期阶段聚焦于补全当前平台已规划但尚未实现的核心分析功能，使平台具备完整的单方案深度分析能力。中期阶段着力于多方案智能比对和知识资产沉淀，将平台从分析工具升级为决策支持系统。远期阶段探索前沿 AI 技术在导弹动力评估领域的深度应用，构建具备自主诊断和设计优化能力的新一代智能评估体系。

三个阶段的划分并非刚性边界，实际开发过程中将根据甲方需求的优先级变化、技术验证的进展和项目资源情况进行动态调整。各阶段内部的具体功能模块亦可根据实际交付节奏灵活拆分和组合。

## 5.2 近期计划——核心分析功能补全

近期计划的目标是补全试验分析模块中已预留但尚未实现的核心功能，使平台在单方案分析的完整性和深度上达到可交付水平。

**压力异常点智能检测。** 当前平台的试验分析模块已实现了基于统计摘要的通道级自动校验（负压检测、超限检测和波动异常检测），能够在宏观层面标记存在问题的传感器通道。近期将在此基础上增加基于时间序列的逐点异常检测功能，实现真正的采样点级异常识别。技术上计划分两步推进：首先实现经典的 3σ（三倍标准差）异常检测算法，将每条压力通道中偏离均值超过三倍标准差的采样点标记为异常点；随后引入孤立森林和 LSTM 自编码器等更先进的异常检测方法，提升对微小振荡、缓变漂移和复合异常模式的识别灵敏度。检测结果将直接嵌入交互式压力曲线图中——异常数据点以红色高亮标注，正常数据点以曲线色保持，使评估人员能够在曲线全貌中直观识别异常发生的时刻、持续时间和严重程度。异常点的统计摘要（数量、时间分布、偏离幅度）将作为新增的诊断信息自动纳入 LLM 分析上下文，使 AI 诊断输入从当前的通道级统计细化到采样点级异常模式，进一步提升诊断结论的细粒度。

**自动化评估报告生成。** 当前平台在各分析模块中已能够分别输出交互图表、AI 诊断结论和结构化统计数据，但这些输出分散在不同的界面卡片和对话记录中，未形成集中的、可供归档的正式评估报告。近期将开发报告生成引擎，实现从分析结果到格式化文档的自动编排。报告模板将包含以下章节：试验基本信息（数据文件名、采样参数、传感器布局说明）、数据质量评估（校验通道总数、异常通道清单及统计、整体数据可信度评级）、关键性能指标（各流道分段的压力均值、特征速度、推力系数等工程参数）、AI 诊断意见全文和交互图表截图附件。报告输出格式优先支持 Markdown 以便于版本管理和协作编辑，后续根据需要扩展 PDF 格式的自动渲染（通过 LaTeX 或 WeasyPrint 等工具链），以适应正式评审场合的格式要求。

**异常检测模型管理界面。** 随着多种异常检测算法的引入，需要为评估人员提供统一的算法配置管理界面。该界面将支持选择和切换不同的检测算法、调整各算法的超参数（如 3σ 的倍数、孤立森林的异常比例、LSTM 的重构误差阈值等），并实时预览参数调整对检测结果的影响。算法选择采用与当前模型管理类似的配置驱动模式，新增算法只需实现统一的数据接口即可注册到选择列表中，无需修改前端代码。

## 5.3 中期计划——多方案智能比对与知识资产沉淀

中期计划的核心目标是将平台从单一方案的分析工具升级为多方案横向比对的决策支持平台，同时建立可持续积累的领域知识资产体系。

**多方案智能比对引擎。** 这是中期计划中工程价值最高的功能模块。当前的仿真分析模块和试验分析模块各自能够独立完成单方案的深度分析，但在多方案比对场景下，评估人员需要人工将各方案的分析结果导出后在外部工具中进行对比。中期将开发专门的多方案比对引擎，在同一界面中支持同时加载多个乙方方案的仿真数据和试验数据，按统一的分析标准批量处理后自动生成多方案横向比对报告。

比对引擎的核心能力包括三个方面。第一，指标自动对齐与差异计算——引擎将从各方案的入口参数报告、壁面力报告、沿程面平均数据和通道统计摘要中提取同名指标，自动完成跨方案的指标对齐，计算各方案相对于参考基准（可以是某一指定方案或所有方案的均值）的差异百分比，并以热力图或雷达图等可视化方式呈现关键指标的多方案对比全貌。第二，异常模式差异分析——引擎将对比各方案中被自动校验标记的异常通道数量和类型分布，识别出哪些方案表现出更优的传感器数据质量和更稳定的燃烧过程。如果两个方案在相同流道位置出现了不同的异常模式，AI 将尝试从燃烧组织方式、传感器安装位置或数据采集系统差异等角度给出可能的解释。第三，综合评分与排序辅助——引擎将基于预设的评估权重体系（各指标的权重可由首席评估专家按需调整）计算各方案的综合得分，生成排序建议，为最终的方案优选决策提供量化的参考依据。

**领域知识库持续建设。** 在平台功能不断完善的同时，RAGflow 知识库的内容建设将同步推进。中期计划将围绕以下方向扩充知识资产：整理和入库更多型号的热试车数据摘要和性能参数基准，形成覆盖不同推力级别、不同推进剂类型和不同燃烧组织方式的性能参数数据库；系统化梳理历次评估项目中积累的专家评审意见和技术决策记录，按"问题—分析—结论—建议"的结构模板整理为评估案例，丰富知识库中的案例推理资源；收录与导弹动力系统设计、试验和验收相关的国军标和行业标准全文或核心条款，建立标准规范知识子库，使评估人员在报告中引用的技术标准能够在知识库中一键检索验证。

**领域模型微调。** 随着人工标注的领域数据达到一定规模（目标为数千条以上经过专家审核的试验数据诊断样本），中期将启动领域模型微调工作。微调基座模型将优先选择已在平台中部署验证过的开源大模型，利用积累的诊断样本数据对模型进行指令微调，使模型在导弹动力领域的专业术语理解、技术规范引用和故障模式判别等方面显著优于通用模型。微调后的模型将与持续更新的本地知识库形成"专业检索 + 专业推理"的双重增强效应，知识库为模型提供经过审核的事实依据，微调模型则更准确地理解和利用这些知识来回答评估人员的专业问题。微调工作的全部流程——数据清洗、格式转换、训练执行、效果评估和模型部署——均在本地 GPU 服务器上完成，训练数据和模型权重始终保留在甲方可控的硬件环境中。

**用户权限与审计日志。** 随着平台在多项目、多评估人员环境中的深入使用，中期将引入基本的用户管理和操作审计功能。主要包括：基于角色的访问控制，区分管理员（可管理模型配置和知识库）、评估专家（可执行全部分析操作和导出报告）和查看者（仅可浏览已有分析结果）三种角色；以及完整的操作日志记录，记录每次数据上传、每次分析执行、每次模型调用的时间、操作者和关键结果摘要，形成满足技术状态管理要求的完整审计追溯链。

## 5.4 远期计划——前沿技术探索与体系升级

远期计划以前沿 AI 技术的工程化应用为导向，探索将平台从辅助分析工具升级为具备自主诊断和辅助设计能力的智能平台的技术可行性。远期计划中的各项技术方向需要更充分的技术验证和数据积累，其具体的实施优先级和时间节点将根据近期和中期阶段的实践反馈进行调整。

**多 Agent 协作诊断框架。** 当前平台的 AI 诊断依赖单一模型完成从数据校验到报告生成的全流程。多 Agent 协作框架将引入多个具有不同专业特长的 AI Agent 协同工作——例如，燃烧稳定性分析 Agent 专注压力振荡的频率和幅值特征、结构力学 Agent 关注壁面力分布与结构承载能力的匹配关系、数据质量 Agent 负责传感器故障模式识别——各 Agent 独立完成各自维度的分析后由协调 Agent 整合形成综合诊断结论。这种松耦合的多 Agent 架构具有两项显著优势：其一，各专业 Agent 可独立进行提示词优化和领域知识注入，不受其他维度分析逻辑的干扰，从而提升单维度分析的深度和准确性；其二，新增分析维度时只需开发和注册新的专业 Agent，无需修改已有 Agent 和协调框架的逻辑，具有良好的可扩展性。技术实现上将调研 LangChain、AutoGen 和 CrewAI 等主流多 Agent 框架，选择与本平台技术栈适配度最高的方案进行集成。

**多模态融合分析。** 当前平台的文本 LLM 诊断和 VLM 图像分析为两条独立的分析链路，各自产出的分析结论需要评估人员人工综合判断。多模态融合分析致力于将文本模态的传感器时序统计数据和图像模态的火焰照片或压力曲线图在同一模型中进行联合推理，实现真正的跨模态信息互补。例如，当压力曲线显示某一时段存在显著的压力振荡时，同时段的火焰图像中可能对应出现火焰颜色或形态的异常变化——单一模态分析可能各自识别出异常但无法建立两者之间的因果关联，而多模态联合推理可以直接输出"压力振荡与火焰形态变化在时间上的同步性表明可能存在燃烧室声-热耦合不稳定性"这类跨模态的诊断结论。技术路线上将首先调研 GPT-4V、Qwen-VL 系列等主流多模态大模型对"数值数据 + 图像数据"联合输入的支持程度，评估是否需要自行开发跨模态注意力融合模块。

**数字孪生自动验模。** 在仿真分析与试验分析均已成熟的条件下，远期将探索仿真与试验的自动比对验证。数字孪生验模引擎将针对同一发动机方案，将 CFD 仿真预测的性能参数（沿程压力分布、推力、壁面热流等）与地面热试车实测的对应参数进行自动比对，量化仿真偏差，并利用机器学习方法分析偏差来源——是网格分辨率不足导致的数值误差、是湍流模型选择不当导致的物理建模误差、还是边界条件设置偏差导致的输入误差。引擎将输出结构化的验模报告，指出仿真模型中需要重点修正的方面和建议的修正方向，辅助仿真工程师高效完成模型的校核与验证工作。

**强化学习驱动的设计参数优化。** 远期将探索将平台的 AI 能力从"评估已有方案"延伸至"辅助优化设计方案"。在建立足够的设计方案—性能指标的映射数据集之后，可引入强化学习或贝叶斯优化方法，在设计参数空间（如喷管扩张比、燃烧室特征长度、推进剂混合比等）中自动搜索满足推力、比冲、燃烧稳定性等多重约束的最优参数组合。每次 AI 生成的候选设计方案可通过 CFD 仿真快速评估其性能指标，评估结果反馈至优化算法以指导下一轮搜索方向，形成"AI 生成设计—仿真验证—结果反馈—优化迭代"的闭环。该方向的实施需要充分的仿真自动化能力和高性能计算资源支撑，属于平台远期能力演进的高阶目标。

**国产化算力与信创环境适配。** 当前平台运行于 NVIDIA GPU 环境，依赖 NVIDIA CUDA 生态。为保障平台在甲方信创环境中的长期可用性，远期将开展国产化算力适配工作，主要包括：将 PyTorch 模型推理迁移至华为昇腾 NPU（通过昇腾 CANN 和 torch_npu 适配层），验证自研 HDF5 后处理引擎在国产操作系统上的兼容性，以及评估国产大模型替代当前商用模型进行诊断分析的可行性。该方向的技术实施节奏将主要取决于甲方信创建设的时间表和国产 AI 软硬件生态的成熟度，平台的技术架构已为此预留了充分的灵活性和可替换性。

**移动端适配与实时告警。** 远期将评估开发移动端轻量版平台的可行性，使评估人员能够通过手机或平板电脑随时查看已完成的评估报告、接收关键分析任务完成的通知推送、以及在紧急情况下快速浏览异常诊断的摘要信息。移动端不追求复刻桌面端的全部分析功能，而是定位为桌面端的"信息终端"——用于结果查看和异常告警，复杂的交互分析操作仍在桌面端完成。实时告警功能将与试验数据的在线接入相配合：当平台未来支持直接对接试验台的实时数据流时，可在试验进行过程中实时运行自动校验规则引擎，一旦检测到预设的严重异常条件，立即通过移动端推送告警通知，使评估人员能够在第一时间介入判断。
