# 阶段 A：重构为 app/ 包 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 把 `demo_server.py` + `src/` 整套后端代码收拢进 `app/` 包，拆分路由、改小写包名、保持功能零变化。

**Architecture:** `app/main.py` 只做 FastAPI 装配（静态挂载 + include_router）；路由按功能拆进 `app/routers/*.py`；原 `src/` 模块整体迁入 `app/core/`；启动入口改为 `run.py`，保留 `demo_server.py` 薄 shim 兼容旧引用。

**Tech Stack:** FastAPI、Uvicorn、Python 3（conda `gy_pytorch`）。

## Global Constraints

- 行为零变化：不新增功能、不改接口路径/签名/响应结构、不改前端行为。
- 静态资源目录 `static/`、数据目录 `data/ Case/ output_plots/ results_png/` 不移动，仅在 `main.py` 挂载，挂载路径与原来完全一致。
- `A00_parameterData.py` 留在项目根，import 写法 `from A00_parameterData import headerIndex` 保持不变。
- 包名 `QJZ_fluent_post` → 小写 `qjz_fluent_post`。
- 工作目录约定：所有启动命令均在项目根 `d:\proj\vc\HYY_PerfomanceAnalysis` 下执行（保证根级模块 `A00_parameterData` 可被 import）。
- 项目非 git 仓库；每个 Task 末尾的提交步骤：若 `.git` 不存在则先 `git init`（提交信息用中文，遵循 `[优化]` 前缀，不含任何 AI 署名）。
