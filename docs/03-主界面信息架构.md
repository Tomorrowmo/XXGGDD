# 主界面信息架构（评估型平台原型）

> 把软件从"单文件分析工具"升级为"多源评估平台"的界面骨架设计。
> 红线：保留现有 UI 布局与左侧导航结构，右侧内容区**渐进替换/增强**，不推倒重来（见 [00](00-需求与定位.md) §6）。
> 核心动线：**入库 → 选对象 → 看对比 → 下钻 → 出报告**（评估为核，见 [00](00-需求与定位.md) §0）。
> 状态：信息架构草案 v0.1。

---

## 0. 三条设计原则

1. **评估优先**：主动线不是"上传→看图"，而是"选要评的对象→看对比→得结论"。分析工具是对比的下钻目的地与证据来源。
2. **壳不变、内容演进**：顶栏标题「组合动力智能评估」、左侧导航、4 套主题、RAGflow 位置全部保留；新能力以**增补导航项 + 演进现有 Tab 右侧内容**的方式落地。
3. **置信度贯穿**：每个数据/结论都带 HIGH/MED/LOW/PENDING 徽章（复用 SimGraph 机制），一眼看出"这条能不能信"。

## 3. 核心屏：📥 数据资源库（入库 + 浏览 + 勾选进对比）

替代现状"临时单文件上传"，是多源对比的前提。

```
┌─ 📥 数据资源库 ─────────────────────────────────────────────────────┐
│ [＋ 批量入库]  [🔎筛选: 单位▾ 批次▾ 类型▾ 工况▾ 物理量▾]  [置信度▾]  │
├─────────────────────────────────────────────────────────────────────┤
│ 分层树 (L1▶L2▶L3)                │  数据卡片列表                     │
│ ▾ 西工大                          │  ┌───────────────────────────┐   │
│   ▾ 2026Q1一轮交付               │  │ ☑ case4  仿真·fluent-hdf5  │   │
│     · case4    [HIGH]            │  │   工况: Ma6-60kPa [HIGH]    │   │
│     · case5    [PENDING⚠]        │  │   收敛✅ 网格12M y+~35      │   │
│   ▶ 2025Q4交付                   │  │   [进对比] [详情] [报告]    │   │
│ ▾ 北航                            │  └───────────────────────────┘   │
│   ▶ 2026Q1交付                   │  ┌───────────────────────────┐   │
│                                   │  │ ☑ 试车03 实验·txt [HIGH]   │   │
│ [已选 3 项 → ⚖️ 去对比]           │  │   工况: Ma6-60kPa …        │   │
└─────────────────────────────────────────────────────────────────────┘
```

要点：
- **批量入库**：目录/多文件拖入 → 后台队列（借鉴 SimGraph 目录监控）→ SHA-256 去重 → simparse 解析 → 自动对齐工况（对不上落 PENDING）→ 生成带置信度的数据卡片。入库进度用现有 SSE 进度条模式。
- **分层浏览**：树 = L1单位 ▶ L2算例/车次 ▶ L3工况；卡片显示概况 + 置信度徽章 + 收敛/网格快照（来自 simparse）。
- **勾选进对比**：跨单位/跨来源多选 → 「去对比」带入下一屏。这是从"单选"到"多源"的关键交互跃迁。
- **PENDING 待办**：对齐失败的项高亮，点进去人工指定工况（喂给映射层）。

## 4. 核心屏：⚖️ 对比评估（平台的心脏）

甲方"谁更优、依据是什么"在这里回答。三种视图对应 L1/L2/L3。

### 4.1 L3 工况点对比（最硬的量化证据 · 默认视图）

固定"工况 × 物理量"，横向摆开各单位值 vs 实验真值：

```
┌─ ⚖️ 对比评估 · 工况点 Ma6-60kPa ────────────────────────────────────┐
│ 视图: [工况点 L3] [算例 L2] [批次 L1]     物理量: 壁面压力 ▾          │
├─────────────────────────────────────────────────────────────────────┤
│ 物理量        实验真值   西工大case4   北航caseX    偏差       判据    │
│ ─────────    ────────  ──────────   ─────────   ───────    ──────   │
│ 壁面压力峰值   3.20 MPa  3.28(+2.5%)  3.55(+11%)  西工大更优  ✅正常  │
│ 室压          2.10 MPa  2.13(+1.4%)  2.31(+10%)  西工大更优  ✅      │
│ 推力          …[HIGH]   …[HIGH]      …[MED]      …          ⚠越界   │
├─────────────────────────────────────────────────────────────────────┤
│ 叠加曲线图(Plotly): 实验/西工大/北航 三线同图 · 各带置信度            │
│ [AI 量化诊断 ↓]  西工大整体偏差 2%内、北航系统性偏高 10%… [生成报告]  │
└─────────────────────────────────────────────────────────────────────┘
```

- 偏差相对实验真值计算；**打分读 sim-knowledge criteria**，触发的判据 intent 可点开看文献/证据（Provenance）。
- 叠加曲线复用现有 Plotly + 2000点降采样能力，但多 trace 来自**不同算例**（现状只能单文件）。
- 「AI 量化诊断」调渊/判据引擎产出可追溯结论；「生成报告」进评估报告屏。

### 4.2 L2 算例对比 / 4.1 L1 批次评级

```
L1 批次评级（甲方汇报层）
┌───────────────────────────────────────────────┐
│         西工大2026Q1   北航2026Q1                │
│ 收敛性     ●●●●○         ●●●○○                  │
│ 网格质量   ●●●●●         ●●●○○                  │
│ 与实验吻合 ●●●●○ 2.1%    ●●○○○ 9.8%             │
│ 综合评级     A-            B                     │
│ [下钻到算例 L2 →] [下钻到工况 L3 →] [导出报告]   │
└───────────────────────────────────────────────┘
```
评级从 L3/L2 加权聚合上来（权重方案待定，见 [02](02-评估元数据模型.md) §7-3）。三层可自由上钻/下钻。

## 5. 演进屏：🔬 仿真分析 / 📊 试验分析（单对象详情 = 下钻目的地）

现有两个 Tab 的能力**全部保留**，定位从"独立工具"变为"从资源库/对比视图点进来的单对象评估页"。

- **🔬 仿真分析**：保留入口参数/壁面力/云图/沿程曲线/VLM 全套；**增强**——接 simparse 补收敛/残差/QOI 面板；去掉 `meshes/1`+`phase-1`+固定字段假设；顶部加"单算例评估结论"（渊 五段式）。
- **📊 试验分析**：保留曲线/LLM诊断/VLM；**重写为可配置**——headerIndex/分隔符/通道命名/单位修正从入库元数据读，不再写死；VLM 图改为用户当前数据（不再 blow.png）；诊断阈值改读判据库；"异常检测"占位按钮落地为判据引擎。

> 即：这两屏是"能力保留 + 去写死 + 作为下钻详情"，不新造。

## 7. 贯穿性 UI 元素

| 元素 | 说明 | 复用 |
|---|---|---|
| 置信度徽章 | HIGH/MED/LOW/PENDING 四色，贴在每个数据/结论上 | SimGraph |
| 证据链弹层 | 点结论 → 源文件+解析tier+判据intent+文献 | simparse+sim-knowledge |
| 自然语言检索 | 顶栏全局框 → 结构化筛选/跳转 | SimGraph NL2Cypher |

| PENDING 待办角标 | 对齐/解析失败项的人工处理入口 | 映射层 |

### 7.1 两个平台级能力的规划

顶栏的两个全局元素不是装饰，是平台级能力，需分阶段建设。

**🔎 自然语言检索 —— 评估元数据之上的三层能力（非搜索框）**

| 层 | 能力 | 例子 | 依赖 |
|---|---|---|---|
| L1 NL→筛选 | 大白话解析成对评估字段（单位/工况/物理量/判据状态/置信度）的过滤条件，跳转筛选结果 | "六院所有 Ma6 且置信度 LOW 的交付" | 评估元数据 schema（[02](02-评估元数据模型.md)）+ SimGraph NL2Cypher |
| L2 NL→聚合查询 | 需要跨库聚合/排序的问题 | "哪家单位推力偏差最小" | + 判据引擎 |
| L3 NL→触发分析 | 延伸到"AI coding 动态生成分析脚本"（会议目标④） | "对比西工大和北航所有工况的壁压偏差，画趋势图" | + AI coding / skills |

- **关键**：L1 前提是先定"可查询字段"——正是 [02] 数据模型在做的事。NL 检索是数据模型的自然出口，不是额外工作量。
- 分阶段：阶段一先做**结构化高级筛选**（顺便固化可查询 schema）→ 阶段二接 NL2Cypher 实现 L1 → 阶段三 L2 聚合 + L3 NL 触发分析。

**📖 Docs / 使用文档 —— 自助文档中心（升级现有 `docs.html` /help）**

| 分区 | 内容 | 阶段 |
|---|---|---|
| 快速上手 | 入库→对比→报告 动线 | 一 |
| **数据规范** ⭐ | 试验 TXT 格式、仿真 HDF5 要求、**工况标识规范** | **一**（入库对齐刚需：各单位须按规范提交，否则工况对不齐，呼应 [02](02-评估元数据模型.md) §3） |
| 功能说明 | 各模块用法 | 一 |
| **评估方法学** ⭐ | 判据/评分/置信度怎么算 | **二**（透明化，取信甲方关键） |
| 术语表 | 名词与数据来源对照 | 二 |
| API / 部署 | 运维 | 二 |

- 演进：阶段三可把文档接入 RAGflow 知识库，实现"问文档"，与平台对话能力打通。
