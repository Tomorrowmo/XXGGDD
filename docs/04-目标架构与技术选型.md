## 1. 关键技术选型

| 决策 | 选择 | 理由 |
|---|---|---|
| Agent 架构 | **DeepAgents**（用户指定） | 主编排+子agent+规划+工具+虚拟FS；渊/析作 subagent |
| 模型接入 | models.json → langchain-openai ChatOpenAI | 复用现有多模型网关，不重造 |
| 数据库 | SQLAlchemy + SQLite（默认，可切 Postgres） | 评估元数据是关系型；SimGraph2 用 Neo4j，本平台按需再上图库 |
| 解析引擎 | **simparse**（直调 sim_parse 包，降级安全） | 覆盖 OpenFOAM/Fluent/HDF5/CGNS，带缓存崩溃隔离 |
| 判据/评分 | **sim-knowledge** criteria + 内置偏差分级 | 护城河判据，读同一份，不写死阈值 |
| 切片可视化 | **SimGraph2 preview_render/simagent_render**（纯标准 VTK） | 四切片 surface/xy/xz/yz，即插即用 |
| 置信度 | HIGH/MED/LOW/PENDING 四级 | 复用 SimGraph 机制，贯穿解析/对齐/测量/评估 |

## 2. 复用 vs 新建（据 SimGraph2 侦察）

- **直接搬**：`preview_render.py`/`simagent_render.py`（四切片，标准VTK）、`export_vtp.py`+`vtp_viewer.html`（交互3D）、`mcp_client.py`（接 simparse stdio MCP）、`coding_tools.py`（沙箱）、`post_data.py`、`physical_mapping.json`、`solver_detector.py`、`openfoam.py`（元数据金矿，作 simparse 补充）
- **改造搬**：`harness.py`（安全拦截→deepagents hook）、`skills.py`（prompt 结构，语义重写）、`graph_tools.py`（置信度 upsert 模型）、`batch_ingest.py`（两阶段入库骨架）
- **全新建**（SimGraph2 没有）：simparse 集成、sim-knowledge 判据体系、多源对比评分、工况对齐、评估元数据模型、DeepAgents 骨架
- ⚠️ **不能搬**：Romtek 私有 C++ VTK 扩展（rtslice/force_moment 等）→ 用标准 VTK 或 simparse qoi 替代
