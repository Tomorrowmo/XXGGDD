# 现状代码能力盘点（真能力 / 写死 demo / 自有可复用资产）

> 本文档基于对后端、前端、以及自有生态三条线的实地代码摸查（2026-07-14）。
> 目的：为重构提供**真实**的现状认知——把"看起来是功能、其实是写死"的部分揪出来，并把可复用的自有资产映射进目标架构。
> 是 [00-需求与定位.md](00-需求与定位.md) 第 2 节"现状能力盘点"的落地依据。

## 0. 一句话结论

- **西工大原型里真正通用可复用的，几乎只有 CFD 后处理（`qjz_fluent_post/*`）+ 多模型网关。试验分析整条链是围绕单一历史工况的写死 demo。**
- **无数据库，纯文件系统；前端无对比 / 无历史 / 无批量 / 无登录 / 无持久化。** 这四大缺失正好对应重构的四大目标方向。
- **底层硬能力我们自己生态里已有 70%+**：`simparse`（解析引擎）+ `sim-knowledge`（评估判据）+ 渊/析（诊断编排）+ `SimGraph`（平台蓝图）。重构应聚焦"拼装 + 补组合动力专域判据"，而非重写底层。

## 2. 前端结构现状

- **已模块化**：`static/index.html`（80 行壳） + 运行时 `fetch` 注入 5 个 `tabs/*.html`（`common.js:loadAllTabPanels`）。`index.monolith.html`（2854 行）是拆分前历史版本，已不被服务；`_split_index.mjs` 是切割脚本（非组件化/打包）。
- **五个 Tab**：🧠 RAGflow（通用对话 SSE + 知识库 iframe）· 🔬 仿真分析（Fluent 后处理全套）· 📊 试验分析（曲线/LLM诊断/VLM）· ✨ 敬请期待（纯占位）· ⚙️ 模型配置（CRUD）。另有独立 `/help` 文档页 `docs.html`。
- **主题**：`data-theme` 4 套（deepspace/minimal/cockpit/sunset），纯 CSS 变量 + localStorage。
- **技术债**：6 处手写 SSE 解析复制粘贴、无 fetch 封装/统一错误/取消；header 表与通道列表全量 `innerHTML` 无虚拟化；无前端框架/打包/依赖管理（CDN plotly/marked）。

### 性能真相（澄清一个误解）
"前端直接渲染大文件"**只在压力曲线一条路径**，且后端 `plotter.py:MAX_POINTS=2000` **已降采样兜底**。真正开销在**后端每次刷新全量重解析文件、无缓存**（`plotter.py:build_pressure_curve`）。前端次要隐患：Plotly trace 数无上限（一键全选通道时）。云图/沿程曲线是**后端 matplotlib 出 PNG**，前端只贴图——重负载在后端计算。

## 3. 写死点 / 技术债清单（重构必清）

1. `headerIndex=10` 全局硬编码 + 逗号分隔假设 —— 试验数据解析绑死单一格式
2. `+0.101325` 大气压修正硬编码 —— 绑定单一压力口径
3. 通道识别只认 `流道\d+` —— 忽略其它传感器
4. 试验 VLM 端点写死 `blow-.png` + 前端硬引用 —— 只能分析一张历史图
5. 前端默认选中流道 22–26 硬编码
6. LLM 诊断阈值硬编码，不可配置
7. 物理常数不一致：`x_slice γ=1.3` vs `symmetry γ=1.4`
8. CFD 字段集固定（云图 P/T/Mach；x-slice 固定 6 项），用户不可扩展
9. Fluent HDF5 结构假设 `meshes/1` + `results/1/phase-1`（单网格单相）
10. 上传进度 job 存进程内存，重启即丢，不支持多进程
11. `llm_client.py` 半死代码：模块级 DeepSeek 客户端已被 `rag_client` 取代，`diagnose_data/chat_stream` 无人调用
12. 遗留离线脚本：`app.py`（Streamlit）、`B01_blowoff_pressure.py`（硬编码绝对路径）、`A00_parameterData.py`（单工况传感器坐标）—— 应剥离
13. **明文密钥入库**：`models.json` 含明文 `api_key`（`token-abc123`/`666666`）；`rag.json` 含分享 auth token + 内网 IP `10.69.140.82`
14. 配置即持久层：`POST /api/models/config`、`/api/rag/config` 直接覆写 JSON，无版本/并发保护
15. 硬编码模型 ID 作默认选中（`deepseek/deepseek-v4-flash`、`local/vllm_Qwen2.5-VL-7B`）散落多处

## 4.5 评估概念的现有雏形（重构不是凭空造，是升级已有的原始形态）

对比评估里的核心概念，并非全部新造——部分在现有代码里**已有写死的、单文件的、非结构化的原始形态**。重构的本质是"升级 + 扩展 + 结构化"，而非推翻。

| 评估概念 | 现有代码里的雏形 | 位置 | 重构方向 |
|---|---|---|---|
| **判据 / 状态判定** | ⭐ 已有写死的原始版：`min<0→严重异常`、`max>10MPa→超限`、`std>mean×2→波动异常` 三条 if-else | `llm_client.py:25-28`（规则）、`llm_client.py:71-88`（`_build_validation_summary` 实现） | 升级为读 `sim-knowledge/criteria.yaml`（知识库、带文献、可配置、多域） |
| **通道统计** | ⭐ 一模一样已有：均值/最小/最大/方差/标准差 | `plotter.py:64-90`（`compute_channel_stats`） | 直接复用，接入评估表 |
| **推力 / 壁面力（QOI）** | 已有：壁面压力+黏性力面积分 | `wall_forces.py:12`（`compute_wall_forces`） | 复用，作为 QOI 来源 |
| **入口参数（QOI）** | 已有：inlet zone 参数提取 | `inlet_conditions.py` | 复用 |
| **"数据是否可信/需人工复核"** | 已有雏形：LLM 自由文本判断（非结构化） | `llm_client.py:33`（system prompt） | 结构化为 HIGH/MED/LOW 置信度 + 评级 |
| **多源对比 / 偏差 vs 真值** | ❌ 无（全程单文件单选，无"真值"概念） | — | 全新建 |
| **结构化评分 / 评级** | ❌ 无 | — | 全新建（评分维度/权重待定，见 [02](02-评估元数据模型.md) §7-3） |
| **置信度分级** | ❌ 无 | — | 全新建（复用 SimGraph 四级机制） |
| **工况点对齐** | ❌ 无 | — | 全新建（[02](02-评估元数据模型.md) §3 心脏） |

> **核心论证**：`llm_client.py` 里那三条写死的校验规则，本质就是"判据"这个概念的原始形态——只是写死在代码里、只有 3 条只管压力、单文件自检。对比评估的重构 = 把这个已有的原始判据机制**升级为知识库驱动、扩展为跨源对比打分、结构化为置信度评级**。这说明重构定位是"把单文件工具里埋着的原始能力，升级成结构化评估体系"，而非另起炉灶。

（注：`B01_blowoff_pressure.py:312` 有"高速相机拍摄结果处理对比"，但那是离线相机图像处理脚本，与"多源数据评估对比"无关。）

## 5. 对重构的直接结论

1. **保留并升级**：CFD 后处理链（去掉单网格/单相/固定字段假设，或直接换 simparse）、多模型网关、文件 CRUD。
2. **重写为可配置**：试验数据解析链（headerIndex/分隔符/通道命名/单位修正全部配置化）、诊断阈值（改为读判据库）。
3. **新建（重构核心）**：数据库持久化 + 评估元数据模型、多源对比、历史/版本、批量入库、（按需）登录会话、智能量化打分、自然语言检索。
4. **优先接入自有资产**：simparse（解析）、sim-knowledge（判据）、SimGraph（架构蓝图）——把重构重心从"造能力"转为"拼装 + 补组合动力专域判据"。
5. **清理**：遗留脚本、半死代码、明文密钥、内存态 job。
