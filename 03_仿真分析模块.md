# 第三章 仿真分析模块

## 3.1 模块概述

仿真分析模块是导弹动力智能评估平台的核心分析引擎之一，面向 CFD（计算流体力学）仿真数据的后处理与智能解读需求，为评估人员提供从仿真原始数据到结构化分析结果的全自动化处理能力。该模块主要服务于对研制单位提交的 CFD 仿真方案进行独立验证和深度审查，确保仿真结果的真实性和可信度，同时支持多个仿真方案的横向量化比对。

在导弹动力系统研制过程中，研制单位通常采用 ANSYS Fluent 等商业 CFD 软件对发动机内部流场进行数值模拟，提交的仿真结果文件（.cas.h5 网格文件和 .dat.h5 求解结果文件）是方案评审的核心技术依据。评估人员需要对这些 HDF5 格式的原始仿真数据进行独立后处理，提取入口参数、壁面力、截面平均参数等关键工程指标，并生成可视化云图和沿程曲线，以验证研制单位所提交的仿真结论是否准确、是否存在选择性展示的情况。传统方式下，这一过程依赖 Fluent 软件本身的后处理功能或商用后处理工具，既增加了软件许可成本，又因操作熟练度和主观判断差异导致不同评估人员得出的结论不一致。仿真分析模块通过自研的轻量级 HDF5 后处理引擎，在完全脱离 Fluent 软件的环境下直接解析原始仿真文件，实现标准化的后处理流程，从根本上解决了上述问题。

## 3.2 自研 HDF5 后处理引擎

仿真分析模块的核心技术基础是一套自主研发的 Fluent HDF5 后处理引擎。该引擎基于 Python 科学计算栈（NumPy、h5py、PyVista、Matplotlib）构建，直接读取并解析 .cas.h5 和 .dat.h5 二进制文件，完全脱离对 ANSYS Fluent 商业软件的依赖，具有部署零成本、运行轻量化、接口标准化的突出优势。

引擎的技术实现分为三个层次。底层为数据解析层，通过 h5py 库直接访问 HDF5 文件内部的网格拓扑数据集和求解结果数据集，包括节点坐标、面-节点关系、面-单元邻接关系等网格数据，以及压力场、温度场、速度场、密度场、壁面剪切应力、组分反应热等求解数据。中层为计算分析层，实现面几何重构（基于节点坐标计算面法向量、面积和面心）、入口边界条件自动识别（通过 zone 名称中的关键字匹配）、压力与黏性力面积分、薄层截面面平均、总温总压和马赫数的热力学关系推导等核心算法。上层为输出层，将计算结果以 JSON 结构化数据、格式化纯文本报告和高质量可视化图像三种形式输出，分别面向程序化调用、人工查阅和报告呈现三种使用场景。

该引擎的关键技术参数如下：

（1）边界拓扑解析：自动识别 .cas.h5 文件中所有 zone 的名称、类型编号、起止面索引和面数，支持 wall（壁面）、symmetry（对称面）、velocity-inlet（速度入口）、pressure-outlet（压力出口）等常见 Fluent 边界类型，为后续的针对性分析提供拓扑基础。

（2）入口参数提取：对所有名称中包含"inlet"关键字的入口边界 zone，读取其邻接单元上的静压、静温、速度和面通量数据，计算入口平均流速、静压、静温和总质量流量等关键工程参数，直接对应发动机设计输入条件，是验证仿真边界条件设置是否正确的重要依据。

（3）壁面力积分：对所有 wall 类型边界 zone，基于面几何信息逐一计算每个面上压力引起的法向力（方向为面内法向的反方向，对应压力作用于壁面的物理方向）和黏性剪切力，并按 zone 汇总三分量合力（Fx、Fy、Fz）。同时输出全部 wall 面的合力总和，直接对应发动机轴向推力、侧向力等宏观力性能指标。

（4）对称面云图生成：将同一流动平面内的所有 symmetry 类型边界 zone 拼接为完整对称面网格，从邻接单元赋入压力、温度和速度场数据，计算马赫数分布，通过 PyVista 可视化引擎生成高分辨率（2560×1440 像素）压力云图、温度云图和马赫数云图各一张。云图采用 jet/hot/rainbow 等工程领域通用的伪彩色映射方案，附带标量色条和标题标注。

（5）沿程截面面平均分析：沿发动机轴向（x 方向）等距划分薄层截面，将各截面范围内的单元按物理量进行面平均计算，生成包含静压、静温、马赫数、总温、总压和释热率六个物理量沿轴向分布的数据表（CSV 格式），并支持对各物理量单独绘制沿程曲线图，横轴为轴向位置（毫米）、纵轴为对应的物理量数值及其单位。

## 3.3 算例管理与上传

模块提供完整的算例生命周期管理功能。算例以工程子目录的形式组织存放于平台的 Case 目录下，每个算例目录包含一对或多对 .cas.h5 和 .dat.h5 文件。平台启动时自动扫描 Case 目录，将已有算例以列表形式呈现给用户。

对于需要新增的仿真方案，模块提供 Web 端算例上传功能。用户指定算例名称后，可将本地的 .cas.h5 和 .dat.h5 文件通过浏览器直接上传至平台服务器。上传过程采用分块写入技术和实时进度推送机制，前端界面同步显示上传百分比和当前处理的文件名。上传过程中平台自动进行文件名校验（仅允许 .cas.h5 和 .dat.h5 后缀）和算例名称安全性校验，防止路径遍历等安全问题。上传完成后，算例文件自动归入对应的 Case 子目录，即可参与后续的自动化后处理流程。

## 3.4 一键式快速分析与分类精细化分析

为了让评估人员能够高效完成仿真方案的后处理审查工作，模块设计了"一键式快速分析"和"分类精细化分析"两种操作模式。

一键式快速分析是推荐的默认工作模式。评估人员选择目标算例后，点击"应用路径并计算入口与壁面力"按钮，系统自动完成以下三项操作：解析边界拓扑列表、提取所有入口边界的参数并生成入口参数报告文本（inlet_parameters.txt）、计算全部壁面的压力与黏性力并生成壁面力报告文本（wall_forces.txt）。三项操作在单次请求中串行完成，整个过程对用户而言仅需一次点击，即可获得算例的边界结构总览和两项核心工程指标数据。入口参数报告和壁面力报告均以制表符分隔的纯文本格式保存于算例对应的 Output 目录下，便于直接在 Excel 或文本编辑器中打开查看和后续处理。

分类精细化分析模式适用于需要深入审查某一特定方面的场景。模块将各项分析功能拆分为独立的操作入口，评估人员可以根据需要单独触发某一项分析任务：

（1）入口参数重计算：当仿真方案的数据文件更新后，可单独重新计算入口参数，生成更新后的 inlet_parameters.txt 报告文件。

（2）壁面力重计算：同样支持独立重新计算，输出更新的 wall_forces.txt 报告文件，便于数据溯源和版本对比。

（3）对称面云图生成：该操作为独立触发，因为对称面网格拼接和 PyVista 渲染的计算量较大，单次耗时相对较长。生成的三张云图（压力、温度、马赫数）自动保存至算例 Output 目录，并在前端界面的对称面云图卡片中以网格布局展示，三张云图并排呈现，点击任意一张可进入灯箱模式查看大图细节。

（4）沿程面平均计算：该操作为所有独立分析任务中耗时最长的一项，因为需要从原始网格中重建体单元中心坐标并进行大量截面搜索和面平均运算。执行完成后，系统自动生成 x_slice_averaged.csv 数据文件和首张物理量（静压）的沿程曲线预览图。评估人员可通过下拉菜单切换物理量，调整横轴和纵轴的显示范围，实时更新曲线图，便于聚焦关注特定轴向区间的参数变化特征。

（5）VLM 云图分析：该操作将对称面的云图（或其他已生成的可视化图像）提交给视觉语言模型进行 AI 解读。评估人员选择目标云图和 VLM 模型后，系统将云图编码为多模态请求发送至模型服务端点，AI 从流场结构特征（激波系、剪切层、回流区等）、关键物理量分布规律、燃烧组织特征和壁面附近流动特征等维度给出专业分析意见。分析结果以流式对话形式实时输出，评估人员可以就 AI 的分析内容进行追问和深入讨论。

## 3.5 前端可视化与交互设计

模块的前端界面采用单页平铺式卡片布局，将算例选择、文件路径、操作按钮和各分析结果展示区组织在同一页面内，评估人员无需切换标签即可总览全部分析流程和结果。

页面顶部为算例选择工具栏，包含算例下拉选择框、加载按钮和上传入口。工具栏下方依次排列文件路径卡片、入口参数卡片、壁面力卡片、对称面云图卡片和沿程面平均卡片，各卡片初始折叠隐藏，仅在对应分析执行完成后自动展开显示结果，保持界面整洁。

壁面力卡片提供了交互式数据筛选功能。壁面力详情表格默认显示所有 wall 面的完整三分量力数据（压力分量、黏性分量和合力），表格顶部设有全壁面合力汇总栏和选中壁面合力汇总栏，评估人员通过勾选或取消勾选表格行首的复选框，可以灵活筛选关注的壁面组合，汇总栏实时更新选中壁面的合力合计值。这一设计对于需要区分不同部件（如喷管内壁、燃烧室内壁、进气道壁面等）受力贡献的精细化分析场景尤其有用。

沿程面平均卡片提供可调参数范围的交互式曲线图。评估人员通过右侧控制面板可以切换物理量类型、调整横轴（轴向位置）和纵轴（物理量数值）的显示上下限，点击"更新曲线图"按钮后前端向服务端发送重绘请求，服务端根据新的参数范围重新生成曲线 PNG 图并返回，实现灵活的数据探索。

对称面云图卡片中的每张云图均支持点击放大，以灯箱模式全屏查看高分辨率原图，便于仔细观察激波结构、边界层分离等微观流动特征。

## 3.6 在多方案比对中的应用价值

仿真分析模块在多乙方方案横向比对场景中具有突出的工程应用价值。

首先，模块通过标准化的后处理流程确保了所有仿真方案在完全相同的计算方法和参数设置下进行分析，消除了因后处理工具差异或操作习惯不同而引入的人为偏差。无论是入口参数的平均方式、壁面力的积分算法还是沿程截面的划分密度，所有方案均采用统一的后处理标准，确保提取的工程指标具有严格的可对比性。

其次，模块输出的结构化数据天然适配量化比对。入口参数报告中的速度、压力、温度和质量流量数据，以及壁面力报告中的各分量力数据，均为制表符分隔的标准化格式，可以直接导入 Excel 或其他数据处理工具进行跨方案的差异计算、百分比偏差分析和排序评比。沿程面平均的 CSV 数据文件更是可以直接在同一坐标系中叠加绘制多个方案的沿程曲线，直观展示不同方案在发动机各轴向区段的性能差异。

最后，VLM 云图分析为仿真比对引入了 AI 辅助的定性判断维度。当多个方案的云图并排对比时，AI 可以辅助识别各方案在激波系结构、射流穿透深度、回流区尺度等流场特征上的差异，并结合知识库中的领域知识判断这些差异对发动机性能的潜在影响，弥补纯定量指标分析可能忽略的流场品质和燃烧组织合理性等定性因素。

## 3.7 技术自主性与扩展性

自研 HDF5 后处理引擎的完全自主性为模块的定制化扩展奠定了坚实基础。由于引擎代码完全自行开发，不依赖任何商业软件的 API 或运行时环境，评估方可以根据实际需求灵活定制后处理算法和分析维度。例如，当需要针对某一特定发动机构型（如带有特殊冷却结构的燃烧室壁面）增加个性化的壁面分区方法时，可以直接在引擎的面几何层和力积分层增加对应的分区逻辑；当需要提取 Fluent 数据集中除当前已实现的压力、温度、速度、释热率以外的其他物理量（如组分质量分数、湍流动能、涡量等）时，只需在数据解析层增加对应数据集路径的读取即可。这种技术自主性确保了平台在面对不断变化的仿真分析需求时，始终具备快速响应和深度定制的能力。
